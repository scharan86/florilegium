# Game Tree

>[!definition]
>It a graph that represents all possible game states within a sequential game that has perfect information. 
>e.g. chess, go, etc.

It's a search tree where: 
1. Each **node** represents a ==game state==
2. Each **edge** represents an ==action==
3. Layers alternate between players
4. Terminal nodes carry 

It's a technique for analyzing adversarial games.

It's just a general structure that can be used for any sequential decision problem with multiple agents. 

A game tree that has been evaluated by the minmax algorithm is called a [[Minimax Tree | minmax tree]].
